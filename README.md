# Каса POS

Проєкт тепер поділений на дві окремі папки:

- `app/` — сама програма каси. Її можна відкрити як просту HTML-сторінку.
- `server/` — окремий Node.js сервер для приймання та збереження товарів, штрихкодів, залишків і чеків.

## 1. Проста програма без сервера

Відкрийте `app/index.html` у браузері. У цьому режимі товари, залишки, налаштування та чеки зберігаються тільки на цьому пристрої у `localStorage`.

> Камера зазвичай не працює, якщо файл відкрито прямо з ZIP або через `file://`. Для камери краще запускати через сервер і відкривати `http://localhost:8787`.

## 2. Окремий сервер для чеків і штрихкодів

1. Встановіть Node.js.
2. Розпакуйте ZIP у звичайну папку.
3. Запустіть сервер із кореня проєкту:

```bash
node server/server.js
```

Або старою командою-обгорткою:

```bash
node pos-server.js
```

4. На цьому ж ПК відкрийте `http://localhost:8787`.
5. На телефоні/планшеті в тій самій Wi-Fi мережі відкрийте `http://IP_ПРИСТРОЮ:8787`.
6. У касі відкрийте **⚙️ Налаштування → Сервер**, увімкніть сервер і введіть адресу типу `http://192.168.1.10:8787`.
7. Натисніть **Зберегти** або **Завантажити**.

Сервер має API:

- `GET /pos-data` — віддає товари, чеки та налаштування.
- `POST /pos-data` — приймає товари, чеки та налаштування і записує їх у `server/pos-data.json`.

## 3. Нові мінімалістичні налаштування

У налаштуваннях більше немає одного великого екрана з усіма полями одразу. Є розділи, які відкриваються натисканням:

- **Основне** — Monobank і час показу ціни.
- **Товари** — додавання товару, список товарів, зміна кількості та видалення.
- **Сервер** — адреса сервера та кнопки синхронізації.

У списку товарів видно:

- штрихкод;
- ціну;
- кількість на складі;
- коли товар востаннє купляли;
- кнопки **Змінити** і **Видалити**.

## 4. Залишки товарів

Коли чек оплачено, програма віднімає куплену кількість зі складу. Наприклад, якщо у чеку 2 молока, то зі складу відніметься 2.

Якщо товару менше ніж 3, зверху показується попередження **“Треба закупитися”**. Кількість у списку товарів підсвічується:

- зелено — 3 або більше;
- оранжево — 1–2;
- червоно — 0.

## 5. iPad iOS 9.3.5

Інтерфейс написаний без сучасного синтаксису JavaScript, щоб краще працювати на старому Safari. Але live-камера на iOS 9.3.5 може не працювати через старий WebKit, тому є ручне введення штрихкоду.
