# Каса POS

Односторінкова каса для сканування товарів, створення чеків, архіву та синхронізації між пристроями.

## Запуск без сервера

Відкрийте `index.html` у браузері. Товари, налаштування та чеки зберігаються локально в `localStorage` цього пристрою.

> Важливо: камера в браузерах зазвичай не працює, якщо відкрити файл прямо з ZIP або через `file://`. Для камери запускайте локальний сервер і відкривайте сторінку через `http://localhost:8787`, `https://...` або з дозволеного локального хоста.

## Запуск із сервером на ПК або Android

1. Встановіть Node.js.
2. Розпакуйте ZIP у звичайну папку.
3. Запустіть сервер:

```bash
node pos-server.js
```

4. На цьому ж ПК відкрийте `http://localhost:8787`.
5. На іншому пристрої в тій самій Wi-Fi мережі відкрийте `http://IP_ПРИСТРОЮ:8787`.
6. У налаштуваннях увімкніть **Сервер** і вкажіть адресу `http://IP_ПРИСТРОЮ:8787`.
7. Натискайте **Зберегти на сервер** або **Завантажити з сервера** для синхронізації товарів і чеків.

Сервер зберігає дані у файл `pos-data.json`. Шлях можна змінити змінною середовища `POS_DATA_FILE`.

## Якщо бачите Git-конфлікт у файлі

Це не помилка програми, а Git-конфлікт після обʼєднання змін. У цьому файлі конфлікт уже виправлено: має залишитися тільки один чистий список кроків без службових маркерів Git-конфлікту.

## iPad iOS 9.3.5

Інтерфейс написаний без сучасного синтаксису JavaScript, щоб краще працювати на старому Safari. Live-камера в браузері на iOS 9.3.5 може бути недоступна через обмеження старого WebKit, тому на екрані сканування є ручне введення штрихкоду як резервний режим.

## Нова окрема версія без конфлікту

Якщо стара сторінка PR не показує файли або GitHub показує конфлікти для `index.html`, відкрийте новий файл `kasa-pos-new.html`. Це така сама каса, але з новою назвою файлу, щоб її було легше знайти й скачати окремо. Для камери все одно краще запускати через сервер: `node pos-server.js`, потім відкрити `http://localhost:8787/kasa-pos-new.html`.

## Якщо GitHub не дає натиснути “Merge pull request”

Повідомлення **This branch has conflicts that must be resolved** означає, що GitHub не може автоматично обʼєднати PR, бо `README.md` або `index.html` змінювалися в двох місцях одночасно. Щоб продовжити, натисніть **Resolve conflicts**, для `README.md` і `index.html` залиште поточну версію файлів без службових рядків конфлікту, потім натисніть **Mark as resolved** і **Commit merge**.

Щоб просто скачати проєкт без злиття PR, відкрийте сторінку репозиторію або гілки та натисніть **Code → Download ZIP**. Після скачування обовʼязково розпакуйте ZIP у звичайну папку перед запуском сервера.
